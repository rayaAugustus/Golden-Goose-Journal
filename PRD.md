# PRD｜金鹅日记（B：成长主导型）

**版本**：v0.1（MVP）

**状态**：可进入原型与开发评审

**作者**：Raya（产品/需求）

**目标平台**：移动端优先（iOS/Android）futter，后续可扩 Web

**核心定位**：成功日记驱动的行动系统，连接梦想与储蓄

---

---

## 1. 背景与机会

许多“记账/理财”产品强调数据与克制，但用户真正缺的是：

- **行动反馈不足**：做了很多但没看到“我正在变好”
- **拖延与空转**：想法多，落地少
- **目标缺乏情绪锚点**：存钱/投资变成痛苦任务而非愿景驱动

《小狗钱钱》启发点在于：

**先建立自信与行动力（成功日记+72小时法则）→ 再把钱系统化（梦想储蓄罐）**。

因此本产品采用 **成长主导（B）**：先让用户每天完成小动作并获得反馈，再温和导入储蓄行为。

---

## 2. 产品目标

### 2.1 业务目标（MVP阶段）

1. 建立稳定留存：用户形成“每天打开 2 分钟”的习惯
2. 形成行动闭环：日记 → 72h挑战 → 完成反馈
3. 实现温和转化：行动完成后，引导“梦想储蓄罐”存入

### 2.2 用户目标

- 每天能清晰看到“我做成了什么”（自信证据）
- 想做的事能在 72h 内至少推进一步
- 梦想不再抽象，储蓄不再痛苦

### 2.3 北极星指标（NSM）

- **连续行动天数**：用户在近 7 天内完成（成功日记 或 72h挑战）的天数

---

## 3. 目标用户与使用场景

### 3.1 目标用户画像（MVP优先）

- 轻焦虑/低动力：容易否定自己、缺乏持续反馈
- 有愿望但不稳定执行：想变好/想存钱/想学习，但难坚持
- 不喜欢复杂记账：需要“轻量可执行”的系统

### 3.2 核心场景

- **夜间**：写成功日记（2分钟完成）
- **白天**：完成/推进一个 72h 最小动作
- **低谷**：抽卡回顾过去成功 → 重启行动
- **奖励**：完成挑战后存一笔小钱到梦想罐（仪式感）

---

## 4. 产品范围（MVP）

### 4.1 必须有（P0）

- 成功日记：每日 3 条 + 连续打卡 + 历史列表 + 成功回顾抽卡
- 72h挑战：新建挑战（含最小动作）+ 倒计时 + 提醒（至少 24h、1h）+ 完成打勾 + 一句话复盘
- 梦想相册：新建梦想（图/金额/日期）+ 梦想墙 + 梦想详情页进度
- 梦想储蓄罐：绑定梦想 + 手动存入 + 存入记录 + 梦想进度联动
- 今日页：将“日记/挑战/梦想/存入”整合成日常入口

### 4.2 暂不做（P1/P2）

- 银行卡/自动扣款/支付闭环（合规与信任成本高）
- 社交分享/排行榜
- 复杂报表与预算
- 多端同步的复杂协作（先保证单端体验）

---

## 5. 信息架构（IA）与导航

### 5.1 底部导航（4 Tab）

1. **今日**（Home）
2. **日记**（Success Journal）
3. **挑战**（72h）
4. **梦想**（Dream + Piggy）

### 5.2 关键入口原则

- “今日页”是默认入口：让用户不用思考下一步
- 日记完成后给“下一步挑战”建议（自然过渡）
- 完成挑战后给“存一笔奖励自己”提示（温和转化）

---

## 6. 核心用户流程（MVP）

### 6.1 每日主流程（留存核心）

1. 打开 App → 进入【今日】
2. 点击【写日记】→ 填写 3 条成功 → 点击【完成】
3. 完成后弹出：
    - 今日连续第 N 天（奖励反馈）
    - “要不要为明天创建一个 72h 最小动作？”（可一键创建）
4. 返回今日：挑战卡片开始倒计时

### 6.2 72h挑战推进流程（行动核心）

1. 在【挑战】页点击【新建挑战】
2. 输入：目标标题 + 最小动作（MVA）+ 自动截止（+72h）
3. 到时提醒 → 用户进入挑战详情 → 点击【完成】
4. 完成后：填写一句复盘（可选）
5. 弹出奖励提示：
    - “给梦想存一笔奖励自己？”（快捷金额 10/20/50/100）

### 6.3 低谷救援流程（情绪复访）

1. 用户在【日记】页点击【成功回顾】
2. 抽取历史成功条目展示（可左右滑动）
3. 提示：“现在做一个 72h 最小动作，帮你把状态拉回来”→ 一键创建挑战模板

## 7. 功能需求详述

# 7.1 模块一：成功日记（P0）

### 7.1.1 功能说明

- 用户每日记录 **3 条成功**（默认 3，后续可配置 3~5）
- 成功允许很小：如“按时起床”“完成一段学习”“少冲动消费”
- 强化反馈：连续天数、抽卡回顾

### 7.1.2 页面：日记-今日填写

**入口**：今日页按钮 / 日记tab顶部

**字段**：

- 成功 1（必填，<=50字）
- 成功 2（必填，<=50字）
- 成功 3（必填，<=50字）
- 心情（可选，1~5）
- 标签（可选，多选：工作/学习/交易/健康/社交/生活）
- 今日学到的（可选，<=80字）
- 明天最重要一步（可选，<=80字）

**交互规则**：

- 成功 1~3 未填齐，不允许提交（提示：写小事也算）
- 支持“快捷示例”按钮：一键填入示例句（降低启动成本）

### 7.1.3 页面：日记-完成反馈弹层

**内容**：

- 连续第 N 天（必显示）
- 今日成功抽卡：随机展示历史成功 1 条（必显示）
- CTA：
    - 主按钮：创建 72h 最小动作（系统预填一个模板，可编辑）
    - 次按钮：稍后再说

### 7.1.4 页面：成功回顾抽卡

- 展示历史成功条目（随机/按标签筛选：P1）
- 操作：喜欢/收藏（P1，可先不做）

---

# 7.2 模块二：72小时挑战（P0）

### 7.2.1 功能说明

任何想做的事，在 72h 内必须推进一个可验证动作（MVA）。

### 7.2.2 页面：挑战列表

- 列表卡片字段：标题、剩余时间、状态（进行中/已完成/已过期）
- 进行中挑战置顶（只置顶 1~3 个，避免压力爆炸）
- CTA：新建挑战

### 7.2.3 页面：新建挑战

**字段**：

- 挑战标题（必填，<=40字）
- 最小动作 MVA（必填，<=80字）
- 关联梦想（可选，下拉选择已有梦想；无梦想可跳过）
- 截止时间（默认当前时间+72h，可手动调但不得超过 7 天：MVP可不开放）

**引导模板（内置）**：

- “发一条消息给某人”
- “查 3 篇资料并写 5 行笔记”
- “存 10 元到梦想罐”
- “完成 15 分钟运动”
- “写 100 字/做一个小提交/写一个函数”

### 7.2.4 提醒策略（MVP）

- 创建成功后默认开启：
    - 距离截止 24h 提醒
    - 距离截止 1h 提醒
        
        （后续可加 48h/6h）
        

### 7.2.5 页面：挑战详情

- 展示：标题、MVA、倒计时、关联梦想
- 操作：完成 / 放弃（放弃需要填写原因：MVP可简化为确认弹窗）

### 7.2.6 完成流程

- 点击完成 → 弹出提交页：
    - 完成证据（MVP先文字<=80字；图片P1）
    - 一句话复盘（可选<=80字）
- 完成后弹出奖励提示：存一笔到梦想罐（见 7.4）

---

# 7.3 模块三：梦想相册（P0）

### 7.3.1 页面：梦想墙

- 卡片：封面、标题、目标金额、截止日期、当前进度（%）
- 排序：按优先级/最近更新（MVP先按最近更新）

### 7.3.2 页面：新建梦想

**字段**：

- 封面图（必填：相册/拍照/默认图库）
- 标题（必填<=20字）
- 目标金额（必填，数值）
- 截止日期（必填）
- 为什么想要它（可选<=120字）

### 7.3.3 梦想详情

- 展示：进度条、累计存入、存入记录入口、关联挑战列表入口（P1）
- CTA：存一笔 / 创建挑战（可选）

---

# 7.4 模块四：梦想储蓄罐（P0）

### 7.4.1 功能说明

以“仪式感存入”承接行动成果：金额小也成立，重点是习惯与反馈。

### 7.4.2 页面：存入弹窗

**入口**：梦想详情 / 今日页 / 挑战完成弹层

**字段**：

- 快捷金额：10 / 20 / 50 / 100（可自定义输入）
- 来源（可选）：工资/副业/节省/奖励/其他
- 备注（可选<=50字）

**规则**：

- 存入后立即更新梦想进度
- 当梦想达成：触发庆祝页（MVP可以简单弹窗）

### 7.4.3 存入记录列表

- 按时间倒序显示：金额、日期、来源、备注

---

# 7.5 今日页（P0）

### 7.5.1 页面结构

- 顶部：今天日期 + 连续行动天数（小字展示）
- 模块卡片：
    1. 今日成功日记（未完成：主CTA“写日记”；已完成：展示3条摘要）
    2. 当前 72h挑战（倒计时 + CTA“查看/完成”）
    3. 当前梦想Top1（进度条 + CTA“存一笔”）
- 底部：快捷按钮（写日记/新建挑战/存入）

## 8. 内容与文案策略（MVP）

- 语气：温柔但坚定（“小事也算”“你已经在变好了”）
- 禁止羞辱式提醒（不使用“你又没完成”）
- 失败处理：过期挑战提示“重新拆一个更小动作？”

---

## 10. 非功能需求（NFR）

- 性能：首屏打开 < 2s（弱网可用）
- 数据安全：本地加密存储（至少对日记文本做加密存储）
- 离线能力：离线可写，联网自动同步（MVP可先本地，云同步P1）
- 通知：可关闭；遵守系统权限规范
- 隐私：默认不公开、无社交

---

## 11. 风险与应对

- **用户压力感**：挑战太多 → 默认限制“进行中挑战最多3个”，并建议最小动作
- **用户写不出来**：日记输入困难 → 提供示例/一键填充模板
- **储蓄引发焦虑**：钱不够 → 强调“象征性也算”，快捷金额可很小（1/5/10 也可配置）
