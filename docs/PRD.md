# Golden Goose Journal 产品需求文档（PRD）

## 1. 产品定位
- **一句话定位**：将记账与成长日记结合，帮助用户通过记录行动与情绪，建立更健康的财富习惯。
- **核心价值**：理财先理心，财富是自信与行动的副产品；让用户在记录金钱流动的同时，追踪心态与成长。

## 2. 目标用户
1. **理财初学者**：有记账需求，但缺乏长期坚持与自我激励。
2. **成长型用户**：希望通过日记记录情绪、行动与自我反馈。
3. **轻量记账人群**：不需要复杂报表，只要可理解的进度与反馈。

## 3. 产品目标
- 提供简单易用的记账体验。
- 将金钱记录与情绪/行动记录关联，形成积极反馈闭环。
- 通过目标追踪增强行动力与自信。

## 4. 需求范围
### 4.1 MVP（P0）
- 记一笔（收入/支出）
- 写日记（文本）
- 创建财务目标（目标金额 + 截止日期）
- 首页概览（余额、当月收支、目标进度）

### 4.2 P1
- 分类/标签
- 日记情绪标记
- 月度预算

### 4.3 P2
- 图表分析
- 目标提醒
- 导出/备份

## 5. 用户流程
### 5.1 新增支出
1. 点击「记一笔」入口
2. 输入金额、类别、备注、时间
3. 保存后回到首页，余额与当月支出更新

### 5.2 写日记
1. 进入「日记」页
2. 输入标题、正文、情绪标签
3. 保存并在列表中展示

### 5.3 创建目标
1. 进入「目标」页
2. 输入目标名称、金额、截止日期
3. 保存后显示进度（已存/目标）

### 5.4 查看概览
1. 首页展示总余额、当月收入、当月支出
2. 显示当前目标进度与最近日记摘要

## 6. 页面结构（信息架构）
- 首页（概览）
- 记账页
- 日记页
- 目标页
- 设置页

## 7. 验收标准（P0）
- **记账**：可新增/编辑/删除一条账目；金额与分类必填。
- **日记**：可创建/查看/按日期过滤；正文必填。
- **目标**：可创建目标并显示当前进度（已存金额/目标金额）。
- **概览**：当月收入/支出正确统计；余额随操作更新。

## 8. 技术栈与部署
- **客户端**：Flutter（移动端优先）
- **本地存储**：SQLite（离线优先）
- **数据同步**：默认不启用云同步，后续按需求评估

## 9. 非功能需求
- 响应时间：主要页面加载 < 2s
- 数据安全：本地优先，支持导出备份
- 隐私：默认本地存储，不开启云同步
