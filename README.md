# Golden-Goose-Journal
市面上的软件大多把“记账/理财”和“个人成长/日记”分开做了，但《小狗钱钱》的核心逻辑恰恰是：理财先理心，财富是自信和行动力的副产品。

## 产品目标与用户画像
- **产品目标**：打造“面向初学理财者的记账 + 成长日记应用”，通过“记录—反思—行动”的闭环，帮助用户建立可持续的理财与成长习惯。
- **核心用户**：18-35 岁、理财认知初级、需要借助轻量引导来建立记账与成长记录习惯的用户（学生/初入职场/自由职业者）。
- **用户痛点**：记账门槛高、日记难坚持、目标缺乏反馈与成就感、对“理财—成长”的关系感知弱。

## MVP 功能清单（含优先级）
| 优先级 | 功能 | 说明 |
| --- | --- | --- |
| P0 | 记账 | 记录收入/支出，支持分类、账户、备注、时间。 |
| P0 | 日记 | 记录心情/反思，支持标签与关键词检索。 |
| P0 | 目标管理 | 设置理财/成长目标，拆解里程碑与提醒。 |
| P1 | 可视化 | 以图表展示收支结构、趋势与目标进度。 |
| P1 | 复盘报告 | 按周/月生成“财务 + 成长”复盘摘要。 |
| P2 | 习惯打卡 | 以轻量打卡强化“记录-行动”闭环。 |

## 关键用户流程 / 页面草图描述
### 1. 记账（添加一笔支出）
- **入口**：底部导航“记账” → “+ 新增”。
- **字段**：金额（必填）、类型（收入/支出）、分类（餐饮/交通/学习等）、账户（现金/银行卡等）、日期时间、备注、标签（可选）。
- **交互**：
  - 金额输入后自动展示“本月同类平均”提示（P1）。
  - 选择分类后显示常用分类快捷入口。
  - 保存后进入当日记账列表并提示“可写一条反思日记”。

### 2. 日记（写一篇成长日记）
- **入口**：底部导航“日记” → “+ 新建”。
- **字段**：标题、正文、心情（表情/分值）、标签、关联账目（可选）。
- **交互**：
  - 支持快速模板（“今天最大的收获”“我为自己做的一件小事”）。
  - 保存后可跳转到“目标管理”关联行动项。

### 3. 目标管理（创建目标）
- **入口**：底部导航“目标”。
- **字段**：目标名称、类型（理财/成长）、目标值/里程碑、截止日期、提醒频率。
- **交互**：
  - 选择目标类型后展示对应模板（如“存下第一笔 1 万”/“完成 30 天记录”）。
  - 完成里程碑后提示“写一条复盘日记”。

### 4. 可视化（查看月度收支）
- **入口**：底部导航“洞察”。
- **页面**：
  - 收入/支出趋势折线图。
  - 分类占比饼图。
  - 目标完成度进度条。
- **交互**：
  - 点击分类可查看明细列表。
  - 时间维度切换（日/周/月）。

## 验收标准
- 记账：可新增/编辑/删除账目；支持按时间/分类过滤；数据持久化。
- 日记：可新增/编辑/删除日记；支持标签与关键词检索；可关联账目。
- 目标管理：可创建/编辑/关闭目标；支持里程碑与提醒配置；展示目标进度。
- 可视化：展示收支趋势与分类占比；支持时间维度切换。
- 复盘报告（P1）：生成周/月摘要并包含可操作建议（如“下周可减少餐饮支出 10%”）。

## 技术栈 / 架构设想
> 目前为规划阶段，以下为建议方案，可根据团队能力与迭代节奏调整。
- **前端**：React + Vite + TypeScript（或 Vue3 + Vite）
- **移动端**：React Native 或 Flutter
- **后端**：Node.js（NestJS）或 Python（FastAPI）
- **数据库**：PostgreSQL（结构化账目/目标）+ Redis（缓存）
- **统计与图表**：ECharts / Recharts
